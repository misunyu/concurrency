SUMMARY

A concurrency bug in Eclipse 3.4.1 Platform UI.

 

DETAILS

Some details can also be found at: https://bugs.eclipse.org/bugs/show_bug.cgi?id=256316

This bug is due to a data race.

 

The boolean "testImage" is initially true. The first thread comes in, goes past line 275 which sets "testImage"

false, but before it gets to line 280 or 283 to set the "imageDesc" the 2nd thread comes in, found

"testImage" false and return "imageDesc" which is still null.
