SUMMARY

A concurrency bug in Eclipse 3.5 Platform Debug.

 

DETAILS

Some details can also be found at: https://bugs.eclipse.org/bugs/show_bug.cgi?id=287834

This bug is due to a data race.

Accessing a hashmap from more than one thread can corrupt its internal structure and it can go into an infinite loop.
