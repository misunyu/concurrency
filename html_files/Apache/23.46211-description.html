SUMMARY

A concurrency bug in Tomcat Fop - Now in Jira, 0.95.

 

DETAILS

Some details can also be found at: https://bz.apache.org/bugzilla/show_bug.cgi?id=46211

This bug is due to a data race.

Use a single reference to the cache during lookup (the cache array could be concurrently replaced).

    final Entry<K,V> removeEntryForKey(Object key) {

    int hash = (key == null) ? 0 : hash(key.hashCode());

    int i = indexFor(hash, table.length);

    Entry<K,V> prev = table[i];

    Entry<K,V> e = prev;

    while (e != null) {

    Entry<K,V> next = e.next;

    Object k;

    if (e.hash == hash &&

   ((k = e.key) == key || (key != null && key.equals(k)))) {

    modCount++;

    size--;

    if (prev == e)

    table[i] = next;

    else

    prev.next = next;

    e.recordRemoval(this);

    return e;

}

    prev = e;

    e = next;

}

    return e;

}
