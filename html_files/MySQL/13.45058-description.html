SUMMARY

A concurrency bug in MySQL-5.0, 5.1

 

DETAILS

Some details can also be found at: https://bugs.mysql.com/bug.php?id=24721

This bug is due to a data race.

Double checked locking: when one thread has the mutex and is initializing structure, the assignment to charset_initialized that it does may be visible to other threads before all of the other writes that it does. This will allow other threads to read invalid data.
