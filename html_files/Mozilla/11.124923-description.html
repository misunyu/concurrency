SUMMARY

A concurrency bug in NSS-Libraries-3.4

 

DETAILS

Some details can also be found at: https://bugzilla.mozilla.org/show_bug.cgi?id=124923

This bug is due to a data race.

 

The dynamic table (oid_d_hash) gets modified (new entries added) when the user loads new crypto modules.  It needs to be protected by a lock.
